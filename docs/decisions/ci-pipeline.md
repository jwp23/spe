# CI Pipeline

Decision: GitHub Actions workflow triggered on push and pull request to `main`. Fast gates (secrets scan via betterleaks, formatting via cargo fmt) run first as parallel jobs. A combined `check` job runs after both gates pass, performing linting (cargo clippy), unit tests (cargo test), and integration tests (cargo test --ignored) in a single compilation. Dependency audit (cargo-audit) and supply-chain checks (cargo-deny, SBOM, Grype) run independently in parallel.

Rationale: CI must run the same checks as the pre-commit hook to prevent drift between local and CI environments. Combining clippy, unit tests, and integration tests into one job avoids three separate compile cycles — the compilation dominates CI time, so one build is much faster than three parallel builds. Fast gates (fmt, secrets) provide early failure before the expensive compilation starts. System utilities (poppler-utils, fontconfig) are installed in the combined check job for integration tests. See ADR-008 for the original parallelization decision.
