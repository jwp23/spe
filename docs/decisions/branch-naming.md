# Branch Naming Convention

Decision: Feature branches follow the pattern `type/short-description` where `type` matches conventional commit types: `feat/`, `fix/`, `chore/`, `docs/`, `refactor/`, `test/`.

Rationale: Aligns branch names with commit message prefixes for consistency. Short, descriptive slugs keep branch lists readable. Examples: `feat/text-overlay-placement`, `fix/pdftoppm-error-handling`, `chore/update-dependencies`.
